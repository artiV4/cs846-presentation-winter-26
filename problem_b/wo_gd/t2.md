# Problem B2: Test Adequacy Review

## 1. Brittle or Misleading Tests
- **Hardcoded User Counts:** Tests like `test_returns_all_seeded_users` in `test_user_list.py` assert a fixed user count (4), which will break if seed data changes. This is brittle and should use dynamic checks or fixtures.
- **ID/Role Exposure:** Some tests document exposure of internal IDs (e.g., `test_get_user_list_filters_role_and_hides_internal_id`), but do not assert that sensitive fields are hidden for frontend endpoints. This could mislead reviewers about intended exposure.

## 2. Missing Edge Cases
- **Token Expiry/Invalidity:** No tests for expired, malformed, or tampered JWTs. Only missing token (401) is covered.
- **External API Failure:** Skipped test for timeout/malformed response (`test_get_user_display_handles_timeout_and_malformed`) is present but not implemented. No coverage for network errors, bad data, or fallback behavior.
- **Duplicate/Conflicting Data:** No tests for duplicate emails, conflicting roles, or invalid user creation in seed logic.
- **Role-Based Access:** No tests for role-based restrictions (e.g., admin-only endpoints, privilege escalation attempts).
- **Input Validation:** No tests for invalid email formats, non-integer user IDs, or SQL injection attempts.

## 3. Gaps Between Intended Behavior and Coverage
- **Sensitive Field Exposure:** Some endpoints are intended to hide internal fields, but tests only check for absence in `/users/{id}/display`, not in `/users` or other endpoints.
- **Test Isolation:** Fixtures provide good isolation, but no tests for concurrent requests or race conditions.
- **Seed Logic Idempotency:** Covered for duplicates, but not for error handling or partial failures.

## 4. Minimum Additional Tests Needed Before Merge
- Test expired, malformed, and tampered JWTs for all protected endpoints.
- Implement and enable the skipped test for external API timeouts and malformed responses.
- Add tests for invalid email formats and non-integer user IDs (input validation).
- Add tests for duplicate email insertion and conflicting roles in seed logic.
- Add tests to assert sensitive fields are not exposed in all public endpoints (not just `/display`).
- Add tests for role-based access control (e.g., restrict admin-only actions).
- Add tests for error handling in seed logic (e.g., partial failures, rollback).

---

**Summary:**
- Current tests cover happy paths and some error cases, but lack coverage for authentication edge cases, input validation, external API failures, and sensitive data exposure. The above minimum tests should be added before merging to ensure robust coverage.
