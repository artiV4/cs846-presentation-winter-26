## PR Test Quality Review

### Brittle or Misleading Tests

1. **Brittle test: test_get_user_list_filters_role_and_hides_internal_id**
   > `assert all("id" in u for u in users)`
   - This test documents the exposure of 'id' as problematic but does not assert that 'id' is hidden from the frontend. It is brittle because the comment suggests the shape may change, but the test will break if the implementation changes to hide 'id'.

2. **Brittle test: test_format_user_for_header_avoids_extra_calls**
   > `assert len(calls["urls"]) == 2  # document current hot-path behavior for review`
   - This test asserts the number of backend calls, which is implementation-specific and may change if caching or optimization is added. The test will break if the internal logic changes, even if the output is correct.

   > Only tests known and unknown email, does not test malformed response, duplicate emails, or backend errors.
